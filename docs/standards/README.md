# Retrovue Standards

## Purpose

This directory defines the shared documentation, testing, and methodology standards used across all Retrovue projects.

## Core Standards

*   **[contributing.md](contributing.md):** The workflow for proposing, reviewing, and maintaining documentation and code, including the AI-assisted development process.
*   **[documentation-standards.md](documentation-standards.md):** The canonical tone, structure, and formatting rules for all documentation.
*   **[repository-conventions.md](repository-conventions.md):** The canonical directory structure, naming rules, and cross-repo alignment.
*   **[test-methodology.md](test-methodology.md):** The testing strategy, validation workflow, and database usage rules.
*   **[ai-assistant-methodology.md](ai-assistant-methodology.md):** How ChatGPT and Cursor collaborate on Retrovue work (contract-first, document-first).
*   **[style-guide.md](style-guide.md):** Deprecated redirect retained for historical references.

## Templates

This directory includes reusable templates for common document types:

*   **[architecture-template.md](architecture-template.md):** For architecture documents.
*   **[changelog-template.md](changelog-template.md):** For changelog entries.
*   **[dev-notes-template.md](dev-notes-template.md):** For development notes and incident reports.
*   **[readme-template.md](readme-template.md):** For component and package READMEs.
*   **[documentation-index-template.md](documentation-index-template.md):** For documentation index pages (docs/README.md style).
*   **[milestone-template.md](milestone-template.md):** For milestone summary documents.
