# ProcessorMetadataContract v0.1

## Purpose

Defines how metadata produced by processors is stored.

---

## Structured Metadata

Metadata required by core system logic MUST be stored in structured tables.

Examples:

- duration_ms
- video_codec
- resolution

These fields are indexed and queryable.

---

## Flexible Metadata

Processor-specific outputs MAY be stored as JSON payloads. Flexible processor outputs MUST be stored in a processor_outputs table containing:

- processor_id
- target_type
- asset_id or media_id
- payload_json
- created_at

Examples of payload content:

- scene classification
- ad break candidates
- speech transcripts

---

## Metadata Ownership

Metadata fields are owned by one of:

**Source**  
Imported during container discovery.

**Processor**  
Generated by metadata processors.

**Operator**  
Modified manually by station operators.

Processors MUST NOT overwrite operator-owned fields. Processors MAY update fields they own when media changes.
